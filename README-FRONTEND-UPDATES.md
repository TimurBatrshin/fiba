# Документация по обновлению фронтенда

## Изменения в API
На бэкенде были внесены следующие изменения, которые влияют на интеграцию с фронтендом:

1. Параметр `registration_open` теперь по умолчанию имеет значение `true`. Если поле не определено или `null`, система считает регистрацию открытой.
2. Добавлен новый статус турнира `REGISTRATION`, который расширяет существующие статусы.

## Внесенные изменения во фронтенд

### 1. Обновлены типы данных
- В интерфейс `TournamentStatus` добавлен новый тип статуса `REGISTRATION`.

### 2. Обновлена логика проверки доступности регистрации
- Создана вспомогательная функция `isRegistrationAvailable`, которая проверяет доступность регистрации на турнир с учетом новых правил.
- Функция принимает во внимание, что если поле `registrationOpen` не определено или `null`, регистрация считается открытой.

### 3. Обновлены компоненты отображения
- Обновлены функции `getStatusText` и `getStatusClass` для корректного отображения доступности регистрации.
- Добавлен учет нового статуса `REGISTRATION` для отображения соответствующих статусов и стилей.

### 4. Обновлена обработка данных с API
- В компонентах, где используется проверка статуса турнира, теперь используется новая логика: `registrationOpen !== false`.
- Обработка ошибок и отображение UI обновлены с учетом новых правил.

## Как проверить
1. Откройте страницу списка турниров и убедитесь, что турниры с незаданным полем `registrationOpen` отображаются как доступные для регистрации.
2. Проверьте, что турниры со статусом `REGISTRATION` корректно отображают статус "Регистрация открыта".
3. Убедитесь, что на странице детальной информации о турнире форма регистрации команды отображается для турниров с незаданным полем `registrationOpen`.
